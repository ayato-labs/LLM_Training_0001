# LLMアーキテクチャ詳細設計書

## 1. 目的
本設計書は、限られたGPUリソース下で最大限のドメイン適応能力と推論/学習効率を発揮するためのLLMプレトレーニングの指針を定めます。

## 2. 設計方針
「絶対的な知能の限界（モデルサイズ）」よりも「GPUリソースの有効活用とコスト効率」を優先します。
近年の中国系モデル（DeepSeek, Qwen等）に見られる、以下の効率化アーキテクチャパターンを主軸に採用します。

1.  **推論/メモリ効率の極大化:** コンテキスト処理量に対するKVキャッシュの削減とメモリ帯域の効率化。
2.  **パラメータの有効利用:** 少ない総パラメータ数で高密度な情報を表現する。
3.  **学習の安定化と高速化:** GPU稼働率を最大化する計算手法の選定。

## 3. アーキテクチャ構成
### 3.1. モデル構造 (Decoder-only Transformer)
- **Attention:** **GQA (Grouped-Query Attention)** を採用。
  - 推論時のKVキャッシュメモリ使用量を大幅に削減し、バッチサイズを向上させる。
- **Positional Embeddings:** **RoPE (Rotary Positional Embeddings)**
  - コンテキストの長さを拡張しやすく、外挿性にも優れる。
- **Activation Function:** **SwiGLU**
  - 標準的なReLUに比べ、表現力と計算効率のバランスが良い。
- **Normalization:** **RMSNorm**
  - LayerNormより計算コストが低く、学習が安定しやすい。

### 3.2. 特化型最適化 (中国系アーキテクチャ模倣案)
- **Attention最適化 (DeepSeekパターン):** 
  - メモリ帯域ネックを解消するため、**MLA (Multi-Head Latent Attention)** の導入を検討する（実装難易度を考慮し、フェーズ2で検証）。
- **データ圧縮:** 
  - 高い圧縮率を持つトークナイザーを採用し、同じトークン数でより多くの意味情報を学習させる。

## 4. 学習と推論の最適化技術
- **FlashAttention-2:** 必須。Attention演算の計算量とメモリ使用量を劇的に改善。
- **Mixed Precision:** **BF16** を使用（ハードウェアが対応している場合。FP8の検討も行う）。
- **Data Pipeline:** 
  - 学習データ（SQLiteから抽出）の高度なクレンジングと重複排除。
  - ストリーミング学習によるメモリ消費の抑制。

## 5. ロードマップ
1.  **フェーズ1: 安定型構成** (GQA + RoPE + SwiGLU + FlashAttention-2) によるベースモデル構築。
2.  **フェーズ2: 高度化** (MLAの導入またはMoE化の技術検証)。
