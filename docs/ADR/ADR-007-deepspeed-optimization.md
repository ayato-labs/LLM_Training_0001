# ADR-007: DeepSpeed を用いた並列学習の導入

- 日付: 2026-06-25
- ステータス: 実装済み

## 背景
限られたGPU VRAM環境下での学習において、計算リソースを有効活用し、学習速度とモデル容量のトレードオフを最適化する必要があった。カスタムの並列ロジック実装は保守コストとデバッグ難易度が高いため、業界標準の最適化手法を採用する。

## 決定
`Hugging Face Accelerate` と `DeepSpeed (ZeRO Stage 2)` を導入し、並列計算とCPUオフロード（オプティマイザ状態のCPU保存）を行う。

## 理由
1. **業界標準の最適化**: `DeepSpeed` は、CPU/GPU間の通信（パイプライン化）が高度に最適化されており、自作ロジックよりも高効率であるため。
2. **保守性の向上**: 学習ロジック（`src/train_model.py`）を一切修正せず、設定（`ds_config.json`, `default_config.yaml`）だけで最適化が可能であるため。
3. **柔軟なスケーリング**: `DeepSpeed` のステージを変更するだけで、将来的なマルチGPU環境への移行も最小限の修正で対応可能なため。
