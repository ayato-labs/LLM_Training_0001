# ADR-0041-github-branch-protection-policy: GitHub上のmasterブランチに対する保護設定の強制とマージ統制の確立

- **Status:** Accepted
- **Date:** 2026-07-19
- **Deciders:** Ayato-labs (ayato-labs)

## Context

本プロジェクトは GitHub 上でパブリックリポジトリとして公開されている。
しかし現在、本番リリース用かつ開発の主流線である `master` ブランチに対してブランチ保護（Branch Protection）が設定されておらず、誰でも直接プッシュや未承認のマージを行うことが可能な状態である。

パブリックコードの安全性および整合性を保ち、オーナーのあずかり知らぬ意図しないコードや悪意ある変更が混入・マージされることを物理的にブロックするため、GitHub 上で厳格なマージ統制とセキュリティ境界を確立する必要が生じた。

---

## Decision

GitHub 上のリポジトリ設定において、`master` ブランチ（および今後追加される本番・準本番ブランチ）に対し以下のブランチ保護ルール（Branch Protection Rules）を強制適用し、オーナーの明示的承認のない一切のマージ行為をシステム制約として拒否する。

### 設定する具体的な保護ルールと設計方針：

1. **プルリクエスト（PR）の必須化（Require a pull request before merging）**:
   - `master` ブランチへの直接プッシュを完全に禁止する。
   - すべての変更はトピックブランチからプルリクエストを作成し、レビューを経ることを必須とする。
2. **オーナー承認の義務化（Require approvals）**:
   - `Require approvals` を有効化し、必要な承認数を **1** に設定する。
   - リポジトリの所有者・オーナーである `ayato-labs` がコードレビューを行い、明示的に「Approve」をしない限り、マージボタンをロックして一切のマージ操作を拒否する。
3. **管理者に対するルールの強制（Include administrators / Do not bypass the above settings）**:
   - リポジトリ管理者（オーナー自身）もこの保護ルールの制限下とし、管理者特権による未承認マージや直接プッシュなどの「ルール迂回（バイパス）」を禁止する。これにより、すべての変更を完全に同一ルートのコードレビューパイプラインに載せる。

---

## Consequences

- **セキュリティとコード整合性の向上**:
  - オーナーである `ayato-labs` の事前合意・監査を経ずに第三者がプルリクエストをマージしたり、直接ブランチを書き換えたりするリスクがゼロになる。
- **ワークフローの標準化**:
  - 管理者自身もルール対象とすることで、操作ミスによる `master` への直接プッシュ（`Direct push to main/develop` 禁止ポリシーの順守）を物理的ガードレールによって未然に防ぐことができる。
