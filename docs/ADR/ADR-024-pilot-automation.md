# ADR-024: パイロット自動化による本番前検証

## ステータス
Accepted

## 日付
2026-07-11

## 決定者
Novel LLM Team

## 背景

本番学習（150Mパラメータ、165時間）を開始する前に、HPOで見つけたハイパーパラメータが本番モデルで機能するかを検証する仕組みが必要だった。手動での確認は運用負荷が高く、失敗時の損失（165時間の浪費）が甚大である。

## 決定

本番学習直前に「パイロット自動検証」を必須ステップとして挟む。

### 仕様
- **実行タイミング**: HPO完了 → Step Law適用 → **パイロット実行** → 本番学習
- **実行条件**:
  - ステップ数: 50ステップ
  - データ量: 0.5% (data_fraction=0.005)
  - チェックポイント保存: なし
  - 評価・アップロード・MLflow: 無効化
  - ログ頻度: 5ステップ毎
- **判定基準**:
  - Loss が NaN/Inf → **即座に失敗、本番中止**
  - Loss > 50 → 警告だが継続
  - 正常終了 → 本番学習へ自動移行

### 実装箇所
- `src/main.py`: `run_pilot_check()` 関数追加、`main()` 内で `train()` 前に呼び出し
- `src/train_model.py`: `pilot_mode` / `data_fraction` 対応、TrainingArguments 動的切替

## 結果

### メリット
- **リスク回避**: 165時間の無駄を数分で検知
- **自動化**: 人間が張り付く必要なし、CI/CD 的運用可能
- **デバッグ容易**: 失敗時、本番規模モデルでのエラーログが即取得可能

### デメリット
- 数分のオーバーヘッド（許容範囲）

## 代替案

1. **手動確認**: 運用負荷高、ヒューマンエラーリスク → 却下
2. **HPOのみで判断**: プロキシモデルでは本番挙動を保証できない → 却下
