# ADR 001: Llamaアーキテクチャへの移行とハイブリッド推論の採用

## ステータス
Accepted

## 背景
RTX 3050 Laptop (VRAM 4GB) という極めて厳しい制約下で、ドメイン特化型小説執筆支援モデルを構築する。当初採用していたGPT-2アーキテクチャは、現代的な最適化技術（GQA等）を十分に活用できず、モデル規模と推論効率のバランスが悪い。また、物語の整合性維持には、コンテキストウィンドウの拡張だけでなく、外部メモリを活用した階層的推論が必要である。

## 決定事項
1.  **モデルアーキテクチャの変更**: GPT-2 から最新の Llama アーキテクチャ (軽量版) へ移行する。GQA (Grouped Query Attention) によるKVキャッシュ削減と、RoPE (Rotary Positional Embeddings) による効率的なコンテキスト処理を実現する。
2.  **階層型物語生成**: コンテキスト拡張の限界を物理技術で補うため、生成時の物語整合性を「長期記憶用外部ストレージ」と「本文生成」に階層分離する。

## 理由
- **Llama移行**: パラメータあたりの性能比（効率性）がGPT-2を大きく上回る。VRAM節約と学習収束の観点から最適。
- **階層型生成**: 4GB VRAMではコンテキスト長512トークンが限界。長編物語では破綻するため、RAGベースの外部記憶により長期間の文脈（キャラ設定やあらすじ）を動的に注入することで、擬似的な長距離記憶を実現する。

## 結果
- 推論時のVRAM消費を抑制しつつ、高い文脈理解力を備えたモデル構築が可能となる。
- コンテキスト長制限をアーキテクチャのみで解決せず、システム設計で解決するアプローチにより、RTX 3050環境での実用性を担保する。
