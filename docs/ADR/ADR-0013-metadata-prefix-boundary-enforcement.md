# ADR-0013: メタデータプレフィックス境界識別の強化（特殊トークンによるセパレータ導入）

## ステータス
Accepted

## コンテキスト

第一回事前学習レポート（2026-06-30）において、150MパラメータLlamaモデルの推論評価で以下の2つの重大なバグが確認された。

### バグ1: プレフィックスと本文の境界線崩壊（TC-05で顕著）
- **現象**: プロンプトプレフィックスの末尾（`タグ: 戦国, 歴史\n\n`）の後に、モデルが他のデータで学習した `主人公, 魔法, シリアスではない...` などのメタタグの羅列をそのまま吐き出してしまう。
- **原因**: 150Mという極小パラメータでは、プレフィックスの「改行と構造」を終わらせて「小説本文を開始する」という構造化ルールを完全に抽象化しきれていない。現在のプレフィックスは `\n\n`（2つの改行）で区切られているが、これが明確な境界シグナルとして機能していない。

### バグ2: 「クノン」支配バイアス現象（TC-01〜TC-04で顕著）
- **現象**: どの作品のプロンプトを入力しても、出力の一部に『ポンコツ魔術師の凶運』の主人公である「クノン」という名前が割り込んで頻出する。
- **原因**: データセットの作品別文字数において『ポンコツ魔術師の凶運』（約617万文字）や『リゼロ』（約1,008万文字）などの一部の巨大作品が学習データ全体の大部分を支配しており、極小作品の語彙が強力な勾配に埋もれている。

### 技術的背景
- **現在のプレフィックス構造**:
  ```
  作品名: {title}
  ジャンル: {genre}
  会話率(全体): {novel_conv:.2%}
  会話率(章): {chapter_conv:.2%}
  感情: {sentiment['label']}
  文字数: {stats['total_chars']}
  タグ: {tags}

  {本文}
  ```
- **問題**: 改行文字のみによる区切りは、150Mパラメータのモデルにとって意味的に不十分。モデルが「プレフィックスの終わり＝本文の開始」を識別できない。

## 意思決定

### 1. 専用特殊トークンの導入
トークナイザーに以下の3つの特殊トークンを追加し、プレフィックス/本文の境界を明示的に定義する。

| トークン | 役割 | 使用箇所 |
|----------|------|----------|
| `<\|start_of_metadata\|>` | メタデータプレフィックスの開始 | プレフィックス冒頭 |
| `<\|end_of_metadata\|>` | メタデータプレフィックスの終了 | プレフィックス末尾（本文直前） |
| `<\|start_of_story\|>` | 小説本文の開始 | 本文冒頭 |

### 2. プレフィックス構造の変更

**変更前**:
```
作品名: {title}
タグ: {tags}

{本文}
```

**変更後**:
```
<|start_of_metadata|>
作品名: {title}
タグ: {tags}
<|end_of_metadata|>
<|start_of_story|>
{本文}
```

### 3. 実装範囲

| ファイル | 変更内容 |
|----------|----------|
| `src/train_tokenizer.py` | 特殊トークンを追加してトークナイザーを再学習 |
| `src/preprocessing/exporter.py` | メタデータプレフィックスにセパレータトークンを埋め込み |
| `src/training/train_model.py` | トークナイザーの特殊トークン設定を更新 |
| `data/tokenizer.json` | 再学習されたトークナイザーで上書き |

### 4. 互換性設計
- 既存の学習済みチェックポイントとは互換性がないため、本変更後はゼロからの再学習が必要。
- レジューム機能はチェックポイントのモデル構造が一致する場合のみ動作するため、再学習開始時に `--resume` フラグは使用しない。

## 結果と影響

### 1. 期待される効果
- **境界識別の確実性向上**: `<|end_of_metadata|>` と `<|start_of_story|>` の2重のシグナルにより、モデルが「メタデータの終わり」と「本文の開始」を明示的に学習できる。
- **メタデータ流出の抑止**: 本文生成時にメタタグが混入するバグの根本的解消。
- **スタイル制御の精度向上**: プレフィックスの構造が明確になることで、推論時のスタイルステアリング（会話率、感情等）の効果が増大。

### 2. メトリクスの定義
本改良の効果は、以下の推論テストTC-05（異ドメイン再現性検証）の改善で定量評価する。
- **成功基準**: プレフィックス末尾のメタタグが本文に流出しない。
- **評価方法**: 10回の生成でメタデータ流出が発生しない割合が80%以上。

### 3. 将来的な拡張
- 将来のモデルスケールアップ（7B等）では、メタデータプレフィックスの有効性がさらに高まる見込み。
- 特殊トークンの追加は、将来的にRAG（外部知識注入）の区切りとして再利用可能。
