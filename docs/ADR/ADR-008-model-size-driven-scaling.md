# ADR-008: モデルサイズ指定による動的スケーリング最適化

- 日付: 2026-06-25
- ステータス: 実装済み

## 背景
スケーリング則に基づくハイパーパラメータ最適化において、現状では固定的なパラメータ推論が行われており、目標とするモデル規模に応じた柔軟な調整ができていなかった。また、データセットの規模とモデルサイズのミスマッチによる過学習のリスクを自動検知する必要があった。

## 決定
目標モデルサイズ（パラメータ数）を入力として受け取り、それに基づいた探索パラメータの決定と、データセット量との比率によるバリデーションを行う仕組みを導入する。

## 理由
1. **モデル中心の開発**: 開発者が「どの程度の規模のモデルを作りたいか」を定義すれば、必要な計算資源と学習レートが自動的に算出されるため、開発体験が向上する。
2. **リスク管理**: 学習開始前にデータセットサイズとモデルサイズの比率を検証し、過学習リスクがある場合に警告を出すことで、無駄な計算資源の消費を防ぐため。
3. **柔軟な探索**: 探索用のパラメータスケール（例: 1/1000）を調整することで、GPUリソースに合わせて探索の負荷をコントロールできるようにするため。
