# ADR-006: スケーリング則に基づく設定ファイルの自動更新とパイプライン統合

- 日付: 2026-06-25
- ステータス: 実装済み

## 背景
スケーリング則（Step Law）によるハイパーパラメータの算出ロジックが存在していたが、算出結果を学習設定（`config.yaml`）に反映させるプロセスが手動であり、再現性の低下やヒューマンエラーのリスクがあった。また、学習パイプラインと算出ロジックが疎結合に設計される必要があった。

## 決定
スケーリング則による計算結果（例: `learning_rate`）を直接 `config.yaml` に自動的に書き込み、その後、学習スクリプトがその設定ファイルを読み込んで学習を実行するパイプラインを構築する。

## 理由
1. **再現性の確保**: 算出された設定をファイルとして永続化することで、後から実験設定の復元が可能になるため。
2. **疎結合なコンポーネント**: 算出ロジック、設定管理、学習実行を独立したプロセスとして分離し、段階的に連結可能な CLI パイプラインを構成するため。
3. **柔軟性とオーバーライド**: 完全自動化しつつも、中間生成物として YAML ファイルを残すことで、必要に応じて人間が直接設定を微調整（オーバーライド）できるようにするため。
