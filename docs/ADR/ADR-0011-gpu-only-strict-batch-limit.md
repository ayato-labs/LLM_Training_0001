# ADR-0011: 4GB VRAM下におけるGPUオンリー学習の強制とWDDMページング回避のための物理バッチサイズ制限

## ステータス
Accepted

## コンテキスト

RTX 3050 Laptop (VRAM 4GB) という極限のハードウェア制約下で、120Mパラメータクラスのドメイン特化LLMをフルスクラッチで事前学習するにあたり、実行速度（スループット）を最大化するためのメモリ管理およびバッチサイズ戦略の決定が必要となった。

これまでに以下の2つのフェーズで致命的な速度低下に直面し、実測データに基づくボトルネックの特定を行った。

### 1. CPUオフロード（DeepSpeed ZeRO-Offload）による速度低下
- **現象**: 120Mモデルの学習時にCPUオフロードを有効化したところ、ステップあたり約45秒（50ステップ完了まで約38分）を要する、あるいは環境によってそれ以上の速度低下を記録した。
- **原因**: 勾配およびパラメータを毎ステップでPCIeバス経由でCPU⇔GPU間転送するため、RTX 3050 LaptopのPCIe帯域（Gen4 x8）が通信ボトルネックとなった。事前学習のスループット維持において、PCIeバス通信を頻発させるCPUオフロードは致命的であると判断し、**「CPUオフロードを一切使用せず、GPU単体メモリ（GPUオンリー）で完結する」**という基本方針を決定した。

### 2. GPUオンリー下でのWDDMページング（共有メモリ退避）による速度低下
- **現象**: GPUオンリーに移行後、物理バッチサイズ `2`（Gradient Accumulation Steps = `5`、実効バッチサイズ `10`）で実行したところ、ステップあたり **約306秒（約5.1分）** という、CPUオフロード時よりもさらに酷い致命的な遅延が発生した。
- **原因**: 
  120Mモデルとオプティマイザの静的なメモリ見積もりは1.56GBであり、一見4GBのVRAMに収まるように見える。しかし、学習時のフォワード/バックワードパスで一時的に発生する「活性化メモリ（Activation Memory）」のスパイクが、物理VRAM上限（4096MiB）を超過した。
  Windows OSのWDDMドライバは、VRAM上限に達したメモリ要求に対し、自動的にメインメモリ（共有メモリ）へ物理ページを退避・復帰させるページング処理を行う。この毎ステップでのPCIe経由のページングが、メモリ割り当てのたびに致命的なオーバーヘッドを引き起こしていた。

---

## 意思決定

物理VRAM内での完全な演算完結（ノンページング）を死守するため、以下の構成変更を決定・実施した。

1. **物理バッチサイズ上限を `2` から `1` へ縮小**
   - VRAM制限が 4.5GB 以下のローカル環境においては、物理バッチサイズの上限を強制的に `1` とする。これによりフォワードパス時の活性化メモリサイズを半減させ、物理VRAMの安全圏（3.2GB以下）に収める。
2. **Gradient Accumulation Steps の倍増による数学的等価性の維持**
   - 物理バッチサイズを半分にする代わり、`grad_accum_steps` を `5` から `10` に自動的に倍増させる。これにより、モデルの勾配更新に使われる実効バッチサイズ（総サンプル数 = `10`）を数学的に100%同一に保ち、学習の安定性と収束性能を維持する。
3. **DeepSpeed による明示的な活性化メモリCPUオフロード（フェイルセーフ）**
   - OS（WDDM）によって自動・暗黙的・かつ制御不能に行われる遅いページングを完全に防止するため、DeepSpeed構成内で `activation_checkpointing`（`cpu_checkpointing: True`、`partition_activations: True`）を明示的に有効化する。
   - これにより、基本はGPUオンリーで高速動作させつつ、万が一コンテキストの局所的スパイク等でVRAM上限を超過した際は、OSのランダムな退避ではなく、DeepSpeedの管理下で一時的に活性化テンソルのチェックポイントのみをCPUに逃がすフォールバック構造を確立する。

---

## 結果と影響

本意思決定を適用し、物理バッチサイズ `1`（grad_accum = `10`）で再起動した結果、以下の劇的な効果が得られた。

### 1. 速度パフォーマンスの劇的な改善（約34倍の高速化）
- **物理バッチサイズ=2 (ページング発生時)**: **約306秒 / step** （50ステップ完了まで約3.7時間）
- **物理バッチサイズ=1 (ページング完全解消時)**: **約8.9秒 / step** （50ステップ完了まで約7分）

物理バッチサイズを1つ減らして勾配累積を2倍にするという、プログラム上の微小な設定変更だけで、WDDMページングが完全に消失し、学習にかかる総時間が **3.7時間から約7分** へと劇的に縮小された。

### 2. ハードウェア制約下の事前学習アプローチの確立
- 4GB VRAMという制約下では、モデル重量だけでなく「一時的なアクティベーション領域」の極小化が生命線となる。
- Windows (WDDM) 環境において、「VRAM警告を無視して物理バッチサイズを増やす」行為は、OSのメモリ退避を引き起こしてスループットを壊滅させるため、**「物理バッチサイズを最小（1）にし、Gradient Accumulationで実効サイズを担保する」**アプローチがローカル事前学習における黄金律（ゴールデンルール）であることが実証された。
