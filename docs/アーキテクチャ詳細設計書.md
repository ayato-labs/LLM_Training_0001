# LLMアーキテクチャ詳細設計書 (V3)

本設計書は、利用可能な計算資源が極めて制限された環境下において、特定のドメイン（小説執筆・分析）に完全特化した超小規模言語モデル（40M〜100Mパラメータ）をフルスクラッチで事前学習するための詳細な技術仕様および実装方針を定義します。

## 1. 背景と目的
現在のLLM開発は大規模な計算資源を前提とすることが一般的ですが、本プロジェクトでは「RTX3050 Laptop (VRAM 4GB)」という極限のハードウェア制約下でのサバイバルをテーマとします。この制約環境下で40M〜100Mクラスのモデルを効率的に事前学習し、小説ドメインにおけるトーン、語彙、キャラクターの整合性を維持する能力の「萌芽」を検証することを目的とします。本フェーズでの知見と技術的実証（PoC）を基に、将来的な最高峰GPUへの本格的な資金・リソース投下の是非を判断します。

## 2. システムビジョンと制約マトリクス
本システムで構築するLLMは、小説の執筆、プロット分析、キャラクターのセリフ生成に特化した「小説特化型マイクロモデル」です。

| 制約項目 | ターゲット仕様 | 技術的アプローチ |
| :--- | :--- | :--- |
| **GPU VRAM** | 4GB (RTX3050 Laptop) | AccelerateによるCPUレイヤーオフロード、Gradient Checkpointing |
| **モデルサイズ** | ~40M-100M パラメータ | 現代的なTransformer構造 (Llamaベース) による高密度パラメータ学習 |
| **データソース** | SQLite（クレンジング済Web小説） | 前書き・後書き・ルビの自動除去、ストリーミング・データローダーによるI/O最適化 |

## 3. モデル構造 (Modern Llama-based Transformer)
総パラメータ数を100M前後に抑えつつ、表現力を最大化するためのアーキテクチャの具体的仕様です。最新のLLMトレンド（Llamaアーキテクチャ）を採用し、安定性と効率を担保します。

### 3.1. 主要コンポーネント仕様
* **Attentionメカニズム: GQA (Grouped-Query Attention)**
    推論時のKVキャッシュのメモリ占有量を削減します。RTX3050環境において、長文脈の処理を可能にするための必須選択です。
* **位置符号化: RoPE (Rotary Positional Embeddings)**
    コンテキスト長の拡張性と外挿性に優れ、小説特有の長期的な文脈維持をサポートします。
* **活性化関数: SwiGLU**
    標準的なReLUやGELUと比較して計算コストと表現力のバランスに優れ、モデルの収束を早めるため採用します。
* **正規化層: RMSNorm**
    計算コストを低減しつつ学習の安定性を担保します。

### 3.2. 推論技術：ハイブリッド推論 (階層型記憶アプローチ)
物理的なコンテキストウィンドウ（512トークン）の制限を突破するため、推論時には「物語メモリ（プロット、キャラ設定、過去の要約）」を外部ストレージから抽出し、動的にプロンプトへ注入する階層型推論を採用します。

## 4. 学習と推論の最適化技術（サバイバルスタック）
VRAM 4GB環境でモデルをOOMさせずに駆動するための協調プロトコルです。

### 4.1. メモリマネジメント
* **Accelerate (CPU Offloading)**
    モデルの重みや勾配を必要に応じてCPU RAMへ退避させ、GPUのVRAMを効率的に使用します。
* **Gradient Checkpointing**
    逆伝播時に中間層を再計算することで、バッチサイズを確保しつつVRAM消費を抑えます。

### 4.2. 計算精度と演算効率
* **Mixed Precision (FP16)**
    16bit浮動小数点数での計算を軸とし、メモリ消費を大幅に低減します。
* **FlashAttention-2**
    Attention演算を最適化し、スピードアップとメモリ節約を同時に達成します。

## 5. データパイプライン
### 5.1. SQLite連携
SQLiteスキーマから学習データローダーを直接生成し、クレンジング済みデータのみをストリーミングで供給します。
### 5.2. トークナイザー設計
小説データで訓練した専用BPEトークナイザーを使用し、圧縮率を最大化してモデルパラメータの効率利用を図ります。

## 6. 成功基準
1. 学習Lossの収束（右肩下がり）を確認すること。
2. 特定の小説のキャラクターの口調や作風の片鱗（単語の選択、特徴的なセリフ回し）が出力に現れること。
この2点が確認できた場合、このアーキテクチャは有効であると判断し、モデルのスケールアップへ移行します。
