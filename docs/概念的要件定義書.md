# 概念的要件定義書

## 1. 目的
本ドキュメントは、小説執筆支援LLM構築プロジェクトにおける意思決定の軸を定義します。技術的な「How（実装方法）」や「What（機能目標）」ではなく、プロジェクト全体を通じた「方針の選定基準」を明文化します。

## 2. 方針の核心（原則）
**「限られたGPUリソースにおける、小説知識密度とキャラクター再現性の最大化」**

本プロジェクトは、計算リソースの潤沢な大規模企業とは異なる制約下にあることを前提とします。したがって、すべての意思決定は「その手法は、リソースあたりの知能・再現性向上に寄与するか？」という問いに依拠します。

## 3. 採用・非採用の基準（意思決定ロジック）

| カテゴリ | 採用するもの（Yes） | 採用しないもの（No） | 理由 |
| :--- | :--- | :--- | :--- |
| **アーキテクチャ** | GPU効率が高い構成 (GQA, RoPE, SwiGLU) | 汎用的な標準Transformer構成 | 計算効率とパラメータ利用効率の差が大きいため |
| **モデルサイズ** | 小中規模 (リソース内で安定稼働するもの) | 極大規模なモデル | 推論速度と学習コストがリソース制約を超えるため |
| **学習戦略** | プレトレーニングによるドメイン適応 | 既存LLMへの単純なファインチューニングのみ | ドメイン知識の密度を深めるためには、根幹からのプレトレーニングが不可欠であるため |
| **Attention** | FlashAttention-2, GQA | 標準的なVanilla Attention | メモリ帯域と計算速度のボトルネックとなるため |
| **検証・実装** | 実用的な安定技術の先行導入 | 複雑かつ実験段階の最適化技術 (現状で) | まずは安定した環境でのドメイン学習を優先するため |
| **データ** | 小説に特化したドメインデータ | 汎用Webクロールデータ | ドメイン特化という目的に対するノイズとなるため |

## 4. 方針に基づく意思決定の考え方
- **リソース対効果:** 新しい技術や手法を取り入れる際は、その導入コスト（実装難易度、計算リソース消費）と、得られる知能向上分を天秤にかけます。リソース浪費の可能性が高いものは、いかに性能が良いとしても現時点では採用しません。
- **「小説」への適合性:** 汎用的な性能向上よりも、小説執筆支援において重要となる「長コンテキストの一貫性」「文体・トーンの再現」「キャラクターの整合性」を直接向上させる要素を最優先します。
- **拡張性 vs 複雑性:** 将来的な性能向上（例: MLAの導入）への道筋は閉ざしませんが、実装を複雑化させるだけの要素は排除します。
